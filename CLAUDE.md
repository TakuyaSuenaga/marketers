# Claude Code ガイドライン

## 🧠 思考の原則
- **前提の明示**: 実装を開始する前に、自分の前提を明確に述べること。確信が持てない場合は必ず質問すること。
- **選択肢の提示**: 複数の解決策がある場合は、勝手に選ばずに提示して議論すること。
- **シンプルの追求**: よりシンプルなアプローチがある場合は提案すること。過剰な複雑化には「押し返す（NOと言う）」こと。
- **不明瞭なら停止**: 指示が曖昧な場合は、何が不明かを特定して確認すること。

## 🛠 コーディングスタイル
- **最小限の変更**: タスクに関連しないコードには一切触れないこと。
- **DRYの維持**: 既存のパターンを再利用し、重複を避けること。
- **型定義**: 型は厳格に。ただし過剰な抽象化は避けること。
- **エラー処理**: 早期リターン（Guard Clauses）を優先し、ネストを浅く保つこと。

## 🔄 コミュニケーション
- **簡潔さ**: 解説は最小限に。コードと重要な判断理由に集中すること。
- **ステップバイステップ**: 複雑な変更は小さなステップに分け、各段階で確認を求めること。
